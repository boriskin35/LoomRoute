# Архитектура экосистемы LoomRoute

Этот документ описывает философию дизайна, механизмы маршрутизации контекста и структуру папок фреймворка **LoomRoute** для ИИ-агентов.

---

## 1. Философия дизайна: Изолированные контексты

Главная идея LoomRoute: это **фильтрация контекста (Context Gating)**.

Большинство ИИ-кодеров начинают ошибаться по мере роста проекта, так как они загружают слишком много нерелевантной информации. Это приводит к перерасходу токенов, путанице и замедлению ответов.

LoomRoute решает эту проблему путем разделения работы на **изолированные среды (окружения)**:
- **Work (`./work/`)**: Специализированная среда для разработки, написания кода, деплоя сайтов и задач для проектов.
- **Individual (`./individual/`)**: Специализированная среда для личного дневника, базы знаний (Второго мозга), рефлексии и фиксации мыслей.

### Ключевые правила:
- **Второй мозг (`individual`) не является корнем** рабочей области. Это всего лишь одна из изолированных сред.
- Корень репозитория выполняет роль **декларативного роутера (маршрутизатора)**. В корне не хранятся файлы проектов или личные данные.

2. **`registry.md` (Реестр)**: Указывает агенту, какая папка среды соответствует ключевым триггерным словам из запроса.

## 3. Структура директорий фреймворка

Скелет рабочей области спроектирован так, чтобы быть чистым и модульным:

```
./
├── CLAUDE.md              ← Корневая конституция (Роль роутера)
├── README.md              ← Описание проекта и быстрый старт на английском
├── README.RU.md           ← Описание проекта и быстрый старт на русском
├── registry.md            ← Таблица сопоставления сред и ключевых слов
├── docs/                  ← Техническая документация
│   ├── ARCHITECTURE.md    ← Архитектура фреймворка (Английский)
│   ├── ARCHITECTURE.RU.md ← Архитектура фреймворка (Русский)
│   ├── METHOD_KARPATI.md  ← Методология рабочих пакетов (Английский)
│   └── METHOD_KARPATI.RU.md ← Методология рабочих пакетов (Русский)
├── skills/                ← Глобальные ИИ-навыки (archive, safe-copy и др.)
├── templates/             ← Глобальные шаблоны (брифы, чек-листы)
├── archive/               ← Глобальный архив выполненных задач
├── work/                  ← Рабочая среда (веб-приложения, боты, проекты)
│   ├── CLAUDE.md          ← Локальные правила рабочей среды (Роль разработчика)
│   ├── registry.md        ← Реестр проектов внутри work/
│   └── skills/            ← Локальные навыки деплоя (Vercel, Surge)
└── individual/            ← Персональная среда (Второй мозг)
    ├── CLAUDE.md          ← Локальные правила Второго мозга (Роль куратора)
    ├── registry.md        ← Реестр заметок внутри individual/
    ├── MEMORY.md          ← Долговременная техническая память среды
    ├── skills/            ← Локальные навыки захвата (note-capture)
    └── inbox/             ← Входящие сырые черновики и файлы
```

## 4. Как добавлять свои слэш-команды

Добавить свои собственные команды для автоматизации рутины (например, деплоя, миграции баз данных или генерации тестов) очень просто:

1. **Создайте файл навыка**: Напишите Markdown-файл (например, `./skills/deploy-aws.md` или `./work/skills/deploy-vercel.md`) с пошаговым алгоритмом, который должен выполнить ИИ.
2. **Зарегистрируйте команду**: Откройте файл `./skills/commands.md` и добавьте строку в таблицу «Command Catalogue», сопоставив вашу команду (например, `/deploy aws`) с путем к созданному файлу навыка.
3. **Запустите команду**: Напишите `/deploy aws` в чате с вашим ИИ-кодером, и он автоматически загрузит и выполнит этот алгоритм.
